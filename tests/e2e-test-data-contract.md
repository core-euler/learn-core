# E2E Test Data Contract

## Цель
Зафиксировать минимальный и стабильный набор тестовых данных для детерминированных e2e сценариев.

## Users
- `new_user`:
  - Нет завершённых уроков.
  - Доступен первый урок первого модуля.
- `mid_user`:
  - Есть частичный прогресс в модуле 1.
  - Есть locked и available уроки для проверки границ.
- `consultant_user`:
  - Завершён минимум один модуль.
  - Режим consultant доступен.

## Content Fixture
- Минимум 2 модуля.
- В каждом модуле минимум 2 урока.
- Контент уроков содержит различимые темы, чтобы проверять границы retrieval.

## Determinism Rules
- Все e2e прогоны стартуют с известного baseline состояния.
- Состояние пользователей пересоздаётся или откатывается перед suite.
- Лимиты для тестового окружения настраиваются предсказуемо для кейсов лимитов.

## Isolation
- Тестовые данные не смешиваются с dev/prod данными.
- Каждый suite использует отдельный namespace/префикс сущностей.

## Failure Forensics
При падении кейса фиксируются:
- user profile fixture,
- content fixture revision,
- commit hash,
- backend contract revision.
