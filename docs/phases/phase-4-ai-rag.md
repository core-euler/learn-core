# Этап 4: AI Modes & RAG Documentation Pack

## Цель
Описать поведение AI в режимах Лекция, Экзамен, Консультант и правила retrieval.

## Входные условия
- Этап 3 завершён.

## Требования
- Три режима AI с чёткими границами контекста.
- Раздельные правила доступа к материалам по режимам.
- Защита от prompt injection на этапе индексации, сборки и ввода.
- Фиксация структуры метаданных для retrieval.

## Компоненты
- AI session domain.
- RAG indexing domain.
- RAG retrieval domain.

## Логика работы
- Зафиксировать контур принятия решения по источникам ответа.
- Определить поведение при вопросах вне контекста.
- Для экзамена выделить жизненный цикл критериев оценки в пределах сессии.

## Критерии готовности
- Описаны режимные ограничения и исключения.
- Описаны политики безопасного контекстирования.
- Привязка требований к тестам отражена в dna-map.
