# Tests — Documentation Contracts (No Implementation)

Этот каталог пока содержит только тестовые контракты и сценарии проверки требований.

Принципы:
- Сначала фиксируется проверяемое поведение.
- Реализация и автотесты появляются позже, после отдельного подтверждения.
- Каждый контракт ссылается на требования из `docs/spec.md` и фазовые документы.

## Структура
- `contracts/` — набор тест-контрактов по доменам.
- `checklists/` — ручные сценарии приёмки по фазам.
- `traceability.md` — трассировка «требование → контракт → фаза».
