# Этап 7: Frontend App Documentation Pack

## Цель
Формализовать фронтенд-слой MVP так, чтобы реализация была однозначной и согласованной с backend-контрактами.

## Входные условия
- Этапы 2–5 (backend-контракты) описаны и стабильны.

## Требования
- UI-структура страниц и навигации согласована с API и статусами домена.
- Все пользовательские сценарии описаны через состояния интерфейса.
- Обработка ошибок, лимитов и блокировок формализована.
- Стриминг-ответы отображаются устойчиво при реконнекте.

## Компоненты
- App shell и маршруты.
- Экран авторизации.
- Экран dashboard и модулей.
- Экран урока с режимами Lecture/Exam/Consultant.
- Клиентский слой запросов и кэширования.

## Логика работы
- Фронтенд использует backend как источник истины по доступу и прогрессу.
- UI не открывает заблокированный контент без серверного подтверждения.
- Любые статусные изменения происходят после подтверждения API.

## Критерии готовности
- Для всех ключевых экранов описаны состояния: loading/success/empty/error/forbidden.
- Есть явная трассировка фронт-сценариев к backend-контрактам.
- Подготовлен документ приёмки фронта для тестирования.
