# Этап 3: Course & Progress Documentation Pack

## Цель
Описать иерархию курса и правила прогресса пользователя.

## Входные условия
- Этап 2 завершён.

## Требования
- Иерархия: курс → модуль → урок.
- Статусы уроков и модулей: locked/available/completed.
- Правила разблокировки уроков и модулей.
- Правила завершения урока через лекцию и экзамен.

## Компоненты
- Домен модулей и уроков.
- Домен прогресса пользователя.
- API-контракты прогресса.

## Логика работы
- Зафиксировать начальную инициализацию прогресса при регистрации.
- Определить единый алгоритм пересчёта статусов.
- Зафиксировать инварианты для недопущения обхода блокировок.

## Критерии готовности
- Прогресс-логика полностью описана.
- Контракты ответов на блокировки описаны.
- Есть тестовые сценарии в виде чек-листа требований.
